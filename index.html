A. Overview

0What is Oracle Function


Oracle Functions is a serverless platform that makes it easy for developers to write and deploy code without having to worry about provisioning or managing compute and network infrastructure. Oracle Functions manages all the underlying infrastructure automatically and scales it elastically to service incoming requests. With Oracle Functions, developers can focus on writing code that delivers business value


1Pay-per-Use


Customers are charged only for the resources used while a function is running. There’s no charge for idle time! This approach is different from the traditional one of deploying code to a user provisioned and managed virtual machine or container that is typically running 24x7 and which must be paid for even when it’s idle. Pay-per-use makes Oracle Functions an ideal platform for intermittent workloads or workloads with spiky usage patterns.

2Open Source


Rather than building yet another proprietary cloud functions platform, Oracle chose to invest in the Apache 2.0 licensed open source Fn Project and build Oracle Functions on Fn. With this approach, code written for Oracle Functions will run on any Fn server. Functions can be deployed to Oracle Functions or to a customer managed Fn cluster on-premises or even o

3Container Native


Unlike most other functions platforms, Oracle Functions is container native with functions packaged as Docker container images. This approach supports a highly productive developer experience for new users while enabling power users to fully customize their function runtime environment, including installing any required native libraries.

If a suitable VCN in which to create network resources doesn't exist already, log in to the Console as a tenancy administrator and under Core Infrastructure, go to Networking and then:

Click Virtual Cloud Networks and choose a compartment.


Click Create Virtual Cloud Network, enter a name, and select the Create Virtual Cloud Network Plus Related Resources option.


See documentation for more details.

4Create IAM policies


Log in to the Console as a tenancy administrator and under Governance and Administration, go to Identity and click Policies and then:

Select the tenancy's root compartment, and create two policies to give access to repositories in Oracle Cloud Infrastructure Registry:

• a policy to give users access to repositories:

Allow group <group-name> to manage repos in tenancy

• a policy to give Oracle Functions access to repositories:

Allow service FaaS to read repos in tenancy

Select the tenancy's root compartment, and create a policy to give the Oracle Functions service access to the network resources that will be created:

Allow service FaaS to use virtual-network-family in compartment <compartment-name>

Select the compartment in which network resources and Oracle Functions resources will be created, and create two policies to give users access to those resources:

• a policy to give users access to network resources:

Allow group <group-name> to use virtual-network-family in compartment <compartment-name>

• a policy to give users access to Oracle Functions resources:

Allow group <group-name> to manage functions-family in compartment <compartment-name>


See documentation for more details.

A. Set up your client

1Set up signing key


Log in to your Development Environment/Terminal and:

Create a hiden directory where you are going to store your private and public keys

$ cd ~

$ mkdir .oci

$ cd .oci

Generate a private key encrypted with a passphrase that you provide by entering:

$ openssl genrsa -out ~/.oci/<private-key-file-name>.pem -aes128 2048

Change permissions on the file to ensure that only you can read it. Choose <private-key-file-name> for your convinience, for instance oci-private-key.pem :

$ chmod go-rwx ~/.oci/<private-key-file-name>.pem

Remember the passpord that you entered and store it in a note. You are going to use it later

Generate a public key (encrypted with the same passphrase that you provided when creating the private key, and in the same location as the private key file) by entering. Remember to substitute the names for private and public key files:

$ openssl rsa -pubout -in ~/.oci/<private-key-file-name>.pem -out ~/.oci/<public-key-file-name>.pem

Copy the contents of the public key file you just created, by entering a line bellow and store it in a note:

$ cat ~/.oci/<public-key-file-name>.pem | pbcopy


Log in to the Console as an API developer, open the User menu () and go to User Settings. On the API Keys page, click Add Public Key. Paste the public key's value into the window and click Add. The key is uploaded and its fingerprint is displayed.



After your API Key is being generated, copy Fingerprint and paste it in a note.

2Set up OCI profile



Log into the OCI Console to access an information your are going to use bellow


Then open the file ~/.oci/config in a text editor. (If the directory and/or the file don't already exist, create them).

$ cd ~/.oci

$ vi config

i - insert text before the current cursor position

:wq - to save and exit

Add a new profile to the ~.oci/config file as follows:

[<profile-name>]
user=<user-ocid>
fingerprint=<public-key-fingerprint>
key_file=<full-path-to-private-key-pem-file>
tenancy=<tenancy-ocid>
region=<region-name>
pass_phrase=<passphrase>


You can find the necessary information for creating a user profile as follows:


See documentation for creating a Profile in the Oracle Cloud Infrastructure CLI Configuration File for more details.




3Install and start Docker


Log in to your development environment as a functions developer and:

Confirm that Docker is installed by entering:

$ docker version

If you see an error message indicating that Docker is not installed, you have to install Docker before proceeding. See the Docker documentation for your platform (for Oracle Linux, see here).

Assuming Docker is installed, go to the Prerequisites section of the Fn Project home page on GitHub and confirm that the installed version of Docker is at least the minimum version specified there. If not, re-install Docker before proceeding.

Launch the standard hello-world Docker image as a container to confirm that Docker is running by entering:

$ docker run hello-world

If you see an error message indicating that Docker is not running, you have to start the Docker daemon before proceeding. See the Docker documentation.


See documentation for more details.

4Install Fn Project CLI


Log in to your development environment as a functions developer and:

Open the Fn Project CLI README.md file on GitHub and follow the appropriate instructions for installing the Fn Project CLI. As a convenient overview, the instructions are summarized below:

• MacOS using Homebrew: Enter:

$ brew install fn

• Linux or MacOS: Enter:

$ curl -LSs https://raw.githubusercontent.com/fnproject/cli/master/install | sh

• Linux, MacOS, or Windows: Download and run the binary from the Fn Project Releases page on GitHub.

Confirm that the CLI has been installed by entering:

$ fn version


See documentation for more details.

5Create and configure Fn Project CLI context


Log in to your development environment as a functions developer and:

Create the new Fn Project CLI context in your .oic directory by entering:

$ fn create context <my-context> --provider oracle

Specify that the Fn Project CLI is to use the new context by entering:

$ fn use context <my-context>

Configure the new context with the OCID of the compartment you want to own deployed functions:

$ fn update context oracle.compartment-id <compartment-ocid>



Configure the new context with the api-url endpoint to use when calling the OCI API by entering:

$ fn update context api-url <api-endpoint>

For example:

$ fn update context api-url https://functions.us-phoenix-1.oraclecloud.com

Configure the new context with the address of the Docker registry and repository that you want to use with Oracle Functions by entering:

$ fn update context registry <region-code>.ocir.io/<tenancy-namespace>/<repo-name>

where <tenancy-namespace> is the tenancy's auto-generated Object Storage namespace string shown on the Tenancy Information page.

For example:

$ fn update context registry phx.ocir.io/ansh81vru1zp/acme-repo

Configure the new context with the name of the profile you've created for use with Oracle Functions by entering:

$ fn update context oracle.profile <profile-name>


See documentation for more details.

6Generate auth token


Log in to the Console as a functions developer and:

Open the User menu () and go to User Settings. On the Auth Tokens page, click Generate Token. 

Enter a meaningful description for the auth token in the Generate Token dialog, and click Generate Token. The new auth token is displayed.

Copy the auth token immediately to a secure location from where you can retrieve it later, because you won't see the auth token again in the Console.

Close the Generate Token dialog.


See documentation for more details.

7Log in to Registry


Log in to your development environment as a functions developer and:

Log in to Oracle Cloud Infrastructure Registry by entering:

$ docker login <region-code>.ocir.io

For example:

$ docker login phx.ocir.io

When prompted, enter the name of the user you will be using with Oracle Functions to create and deploy functions, in the format <tenancy-namespace>/<username>. For example, ansh81vru1zp/jdoe@acme.com. Check the pictures on the right hand side to see where to find tenancy-namespace and username.

When prompted for a password, enter the user's Oracle Cloud Infrastructure auth token that you created in a previous step #6.

You're now ready to start creating, deploying, and invoking functions.


See documentation for more details.

B. Create, deploy, and invoke your function

1Create your first application


Log in to the Console as a functions developer and:

Go to oracle Functions landing page at https://console.us-ashburn-1.oraclecloud.com/functions. Select the region you intend to use for Oracle Functions(recommended to be the same region as the Docker registry specified in the Fn Project CLI.)


Select the compartment specified in the Fn Project CLI context. Under Applications select Create Application and fill in the fields with NAME, VCN, and SUBNETS information as it shown on the right side picture.

• helloworld-app as the name for the new application. You'll deploy your first function in this application, and specify this application when invoking the function.

• The VCN and subnet in which to run the function.

Click Create.


See documentation for more details.

2Create your first function


Create a folder where you are going to save function related files. Use the name for the folder as the name you used Fn Project CLI.

$ mkdir helloworld-app

$ cd helloworld-app


Create a helloworld java function by entering:

$ fn init --runtime java helloworld-func

A directory called helloworld-func is created, containing:

• a function definition file called func.yaml

• a /src directory containing source files and directories for the helloworld function

• a Maven configuration file called pom.xml that specifies the dependencies required to compile the function

• in a scope of this tutorial, you create java function. Yet, you can create function type based on your needs.


See documentation for more details.

3Deploy your first function


Log in to your development environment as a functions developer and:

Change directory to the helloworld-func directory created in the previous step:

$ cd helloworld-func

Enter the following single Fn Project command to build the function and its dependencies as a Docker image called helloworld-func, push the image to the specified Docker registry, and deploy the function to Oracle Functions in the helloworld-app application that you created earlier:

$ fn deploy --app helloworld-app

(Optional) Confirm that the helloworld-func image has been pushed to Oracle Cloud Infrastructure Registry by logging in to the Console as a functions developer. Under Solutions and Platform, go to Developer Services and click Registry. Choose the registry's region, then click the name of the repository you specified in the Fn Project CLI context to see the helloworld-func function within it.

(Optional) Confirm that the function has been deployed to Oracle Functions by logging in to the Console as a functions developer. Under Solutions and Platform, go to Developer Services and click Functions. Select the compartment you specified in the Fn Project CLI context, then click the helloworld-app on the Applications page to see that the helloworld-func function has been deployed to Oracle Functions.


See documentation for more details.

4Invoke your first function


Log in to your development environment as a functions developer and:

Invoke the helloworld-func function in the helloworld-app that you created earlier by entering:

$ fn invoke helloworld-app helloworld-func

The 'Hello World !' message is displayed.

Congratulations! You've just created, deployed, and invoked your first function using Oracle Functions!


See documentation for more details.

D. Create, deploy, and call your API

1Define your first API deployment


Define details of the API deployment in a JSON file, in the format:

{
  "displayName": "acme-api-deployment",
  "gatewayId": "<gateway-ocid>",
  "compartmentId": "<compartment-ocid>",
  "pathPrefix": "/v1",
  "specification": {<api-specification>}
}


where:

• <gateway-ocid> is the OCID of the gateway you created in which to create the new API deployment.

• <compartment-ocid> is the OCID of the compartment in which to create the new API deployment.

• <api-specification> is a simple API specification. At a minimum, include a {routes:} section containing a path, one or more methods, and the definition of a back-end. The example below assumes you have already deployed a Hello World function to Oracle Functions.

For example:

Download File


In downloaded file you need to change functionId

Go to oracle Functions landing page. Fund your newly creayed function and copy its OCID. For your reference, check the pictures on the right hand side.


Create an API deployment folder apic in the directory where you stored your helloworld-func

$ cd helloworld-app

$ mkdir apis

$ cd apis

Save the JSON file as helloworld-func.json in apic folder. You'll use this file in the next step to create the API deployment.


2Create your first API deployment


Create the API deployment by specifying the JSON file containing the API definition (in our case its helloworld-func.json) in a POST request to the API Gateway REST API:

$ oci-curl apigateway.us-phoenix-1.oci.oraclecloud.com post ./helloworld-func.json "/20190501/deployments"



The response to the request includes an OCID as the id of the new API deployment.

Use the API deployment's OCID to confirm that the API deployment is active:

$ oci-curl apigateway.us-phoenix-1.oci.oraclecloud.com get "/20190501/deployments/<deployment-ocid>"

When the API deployment is active, go on to the next step.


3Call your first API


Open a terminal window and call the API by entering:

$ curl -k -X GET https://<gateway-hostname>/v1/hello/

where <gateway-hostname> is the hostname of the gateway you created on which to create the new API deployment. For example, laksjd.apigateway.us-phoenix-1.oci.customer-oci.com


You can use Terminal, Web Browser, or Postman to call your API



Congratulations! You've just created your first API gateway and API deployment, and called your first API using the API Gateway service!


 You're done!
